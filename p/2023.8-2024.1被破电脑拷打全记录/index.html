# 垃圾鼠标

深受其害: 9.2-2023-10-20

‍

大概是鼠标的省电模式的问题, 导致长时间不操作(5s)后发生时延;

但是经过一段时间的使用后发现, 更有可能的系统的忙碌, 鼠标进程的优先级被抢占导致的操作迟缓

不知道最后的结果是什么: 我确信, 只要把电脑重装后测试即可

后来发现不是鼠标的事情, 是电脑服务设置的问题.

‍

总之不会再买这家的无线蓝牙静音鼠标了(说的就是你, 英菲克!)

超了, 最后一个鼠标才用 8 个月又坏了, 经典只能用一年然后报废, 因为坐车的时候没有装盒子的话就会导致磕坏点击板或者是平时放书包没保管好就会磕碰到, 可靠性 40%的废物

# 垃圾网卡

网卡只要切换一下网络就会找不到 WIFI 网络…

使用修改网卡的 VMware 导致 Wifi 适配器被禁用

‍

深受其害: 9.2 - 2024. 1.30 过年前才修好, SK 你 TM 是真的能忍啊

算了罢, 更新全部失败, 驱动坏了两个, 能用就行吧

为了保证当前虚拟机的正常运转, 什么都可以忍受

2023-10-28 现在大概只能去服务列表里使用重启 WlAN autoConfig 服务来不重启完成连接了…

‍

大概虚拟机的网课配置有关(关于 IP 设置等,当初为了调试为非桥接模式废了很大的功夫但是收效甚微, 大概是因为外面还套了一层 VPN 的原因, 流量走的不是固定的 IP 地点)

24 年的我要说, 你太聪明了, 完全正确, 不愧是 SK!

最后的结果就是网卡打开后系统的 Wifi6 适配器就没了, 再也打不开只能按照下面强制重启 Wifi 设置, 运气好的时候使用 Wifi 控制中心的 Wifi 图标禁用就能解决. 运气不好就白白浪费生命中的一分钟去乱点一些没用的 Button 最后还是走重启流程, 就连重启有时候都无法解决!!! 烦内!!!

发生的频次: 频繁切换校园网或者是流量或者 Wifi 会导致发生概率大大增加,解决长时间待机后连不上网

因此之后我经常哪怕学校网再烂也不去切换为自己的网, 就是怕切回来后切不回去了(直接 WLAN 可选列表干没 – 你找不到 Wifi 网络… 😋)

大概也叠加了 Win 之前的老 bug…之前大概 3 个月才能非常倒霉的来一次, 甚至还要让家里的 Wifi 背锅…

看来是 Win10 底层的东西了, 还是尽量不要动系统了, 没有靠谱的软件的话

这里事实上我为了达到所谓的超级续航, 封禁了系统一大批功能, 可能也间接导致了这个网络的问题, 现在是没精力鼓捣了, 老老实实用罢. 也不追求性能和续航了反正从学校 run 了整天插电也不用考虑续航了(大概)

然后呢? 我现在在图书馆, 电池只能用 4h… 续航还是很重要的拜托

这一连串, 就是反复打脸和回旋镖的过程…吸取教训, 下次遇到这样的问题绝对不要忍了, 就好像三蛋在台上呆了 50 年, 点数亏麻了太子都熬死了, 还不知道陛下请上马一下…

‍

大佬的排查笔记CSDN, 翻遍全网没几个出问题的, 大概是其他修改软件又把注册表修坏了

其中尝试了一个月可用的只能手动重启服务: 任务管理器找 Wlan AutoConfig (但每次去任务管理器还是很麻烦) 设置自动任务 - 当崩溃时重载也不行 -

这里提到了之前 8 月份重装的策略, 现在想起了一个点, 就是重装前已经不堪重负的旧系统我为了提前演练一下 VMware 的操作(例如创建虚拟机和安装 JDK), 先装了, 之后没有卸载还是卸载了忘记了 ( 反正根据广泛流传的关于不能卸载 VMWare 的传言来看, 当时或多或少受到了这个影响), 然后重装选择的是部分重装, 系统设置什么的导入也存在问题, 当时就觉得不对劲了…

(本人一直有鼓捣系统服务等各种原有配置的习惯) 后来装了 VMware 之后情况就急转直下了, 特别是选择固定 IP 的时候 (之后一直没能设置上, 导致每次我重启之后都要把 Linux 系统 IP 对应的一个个子节点重新 ifconfig 然后记录 Ip 存到 Hosts 文件里…浪费 8 分钟/次, 总共持续了 6 周, 就是浪费了人生中的一小时!!!NO! 为什么我不去重装?! 为什么?! 因为自己鼓捣的系统白白重装了没了很心痛啊… 但是现在我已经成了没有感情的重装机器, 十次也好, 百次也罢, 只要需要我重装, 我一定会做的(甚至给自己写了一份完整的灾难处理重装手册 2333)

‍

‍

没用的操作, 一两次还行, 后面玄学基本没用, 不如直接关闭服务来得快

**以管理员身份运行命令提示符输入**​netsh winsock reset 后重启

​Win+Q​系统配置，选择服务, 到服务中的 network setup service，将其左边的勾取消掉，保存，选择退出而不重新启动。在进入服务里启用它，就不会自动关闭了。

# 驱动更新被拷打

‍

‍

驱动更新导致连不上网 - 仍然在设置此设备的类配置

更新网卡后有驱动问题, Windows 仍在设置此设备的类配置

‍

网友介绍是注册表问题, 大概是 VMware 不正确卸载导致的问题. 害人不浅, 绝对不会用了…需要重建注册表, 那么我用 Windose 管理工具来烩一烩它

‍

‍

看来还是老实恢复旧驱动好

驱动真就难倒英雄汉了, 没有网络真的是寸步难行

‍

现在打算应急重装一下. 保留思源内容

无妨, 正好重装系统, 现在先尝试一个下午看看 (强装镇定)

‍

结果之后就发生了 12 - 24 著名小丑事件: 装了鼠标新驱动导致鼠标脱手不响应事态结束了, 当时装完了鼠标驱动修完了我还是不满足, 希望用同样的方法把 WLan 驱动也修好, 解决所谓 WLAN 崩溃问题… 当时装好后就完全不能联网了, 很急当天晚上, 第二天紧急重装了. 心想反正没什么事了也就写个 CSS & LDBMS 小组作业加上期末复习而已, 还运转的过来(结果真的运转不过来了, 后来间接导致了 1.9 滑铁卢小丑事件和 1.15 死线战士俩个恶性事件(后话)

为什么说是小丑事件? 因为装完第一天我拍手叫好! 网络? 启动! 鼠标? 启动! 好起来了…结果第二天鼠标又烂回去了, 不跟手还卡…网卡第三天变回去, 一旦换网络 80%断网… 结果为了期末俩大项目和期末复习还不能重装, 就这么熬到了 1.15 终于开始完全电脑搬家彻底扫除影响, 选择了从云端获取全新的 Win 系统选项, 同时抹除了磁盘所有玩意(没错, 包括你! VMware), 到今天 2024-03-18 终于可以说是阴霾散去了 OTZ
